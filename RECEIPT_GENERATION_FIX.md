# Receipt Generation Bug Fix - March 15, 2026

## Issues Found and Fixed

### 🔴 Issue 1: Non-existent Model Method Calls
**Location:** `zeliaoms/store/receipt_generator.py` - Lines 68, 91, 193

**Problem:**
The code was trying to call methods that Django doesn't auto-generate:
- `order.get_store_display()` - **Doesn't exist**
- `order.get_customer_category_display()` - **Doesn't exist**  
- `order.get_paid_status_display()` - **Doesn't exist**

This caused `AttributeError` when generating receipts.

Lookup methods like `get_*()`_display()` are ONLY auto-generated by Django for ChoiceField when the method is called on a model instance that HAS that field. Since these are called on `order` object, they don't exist.

**Fix:**
Manually resolve the display values using the model's `*_CHOICES` tuples:

```python
# Before (BROKEN):
['Store:', order.get_store_display() if hasattr(order, 'get_store_display') else order.store]

# After (FIXED):
store_display = dict(order.STORE_CHOICES).get(order.store, order.store) if hasattr(order, 'STORE_CHOICES') else order.store
['Store:', store_display]
```

Applied to:
- `STORE_CHOICES` → `store_display`
- `CATEGORY_CHOICES` → `category_display`
- `PAID_STATUS_CHOICES` → `paid_status_display`

### 🔴 Issue 2: Unsafe Null/None Handling
**Location:** `zeliaoms/store/receipt_generator.py` - Lines 68, 183

**Problem:**
- Line 68: `order.sales_person.get_full_name()` crashes if `sales_person` is None
- Line 183: Calculating subtotal could fail if order items have null values

**Fix:**
```python
# Sales person with null safety
('Sales Person:', 
  order.sales_person.get_full_name() 
  if order.sales_person and hasattr(order.sales_person, 'get_full_name') 
  else (order.sales_person.username if order.sales_person else 'N/A')
)

# Subtotal with type safety
try:
    subtotal = sum(Decimal(str(item.line_total or 0)) for item in order.order_items.all())
except (ValueError, TypeError):
    subtotal = Decimal('0')
```

### 🔴 Issue 3: Blob Response Error Handling
**Location:** `frontedapp/src/api/orders.ts`

**Problem:**
When PDF generation fails, the error response is a JSON blob, not text. The API client's error interceptor tries to parse it as JSON but can't properly extract the error message.

**Fix:**
Added blob detection and parsing in `downloadOrderReceipt`:
```typescript
export async function downloadOrderReceipt(id: number): Promise<Blob> {
  try {
    const { data } = await apiClient.get(`orders/${id}/download_receipt/`, {
      responseType: 'blob',
    });
    return data;
  } catch (error: any) {
    // If error is a blob, parse it as JSON
    if (error?.response?.data instanceof Blob) {
      try {
        const text = await error.response.data.text();
        const errorData = JSON.parse(text);
        const enhancedError = new Error(errorData.error || 'Receipt generation failed');
        (enhancedError as any).response = { data: errorData };
        throw enhancedError;
      } catch (parseError) {
        throw new Error('Failed to generate receipt. Please check your order data.');
      }
    }
    throw error;
  }
}
```

## Common Scenarios That Caused Errors

### 1. Customer with Null Sales Person
**Scenario:** Admin creates an order with no salesperson assigned
**Error:** `AttributeError: 'NoneType' object has no attribute 'get_full_name'`
**Fix:** Check if `sales_person` exists before calling methods on it

### 2. Missing Choice Display Logic
**Scenario:** Receipt tries to display customer category
**Error:** `AttributeError: 'Order' object has no attribute 'get_customer_category_display'`
**Fix:** Use `dict(Order.CATEGORY_CHOICES).get(category_value)` instead

### 3. Decimal Conversion Errors
**Scenario:** Order item with malformed price data
**Error:** `TypeError: unsupported operand type` or `ValueError: couldn't convert string to Decimal`
**Fix:** Always wrap in try-catch and ensure Decimal conversion

### 4. Empty Order Items
**Scenario:** Creating receipt for order without items
**Error:** `ZeroDivisionError` or incorrect totals
**Fix:** Handle empty sequences gracefully with `sum()` and default values
